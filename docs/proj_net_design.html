ProjNet — One-Shot Structured-Light Correspondence

Design document · lux/proj_net.py · scripts/train_proj_net.py · scripts/predict_proj_net.py

Task: from a single camera capture of a projected M-array pattern (marray/cap_pat_00.png), predict — per camera pixel — the projector coordinate (u, v) that illuminated it (gt_proj.npy), plus a validity mask for pixels the projector cannot see (shadow, out-of-frame, no surface). Dense correspondence from one shot is what makes single-frame metric depth possible: u + calibration → triangulated depth.

input 1×H×W grayscale output 99×H×W (60+36 bins, 2 offsets, validity) 7.85 M params ~300 px receptive field val median |Δu| ≈ 10.5 px @ epoch 10

1 · Why this can work one-shot: the M-array pattern

The projected pattern (patterns/marray, generated by scripts/gen_patterns.py) is a binary grid of 4 px cells (480×270 cells at 1920×1080) constructed so that every 5×5-cell window is globally unique — verified exhaustively: 126,616 windows, zero duplicates, in a 225 code space. A ~20×20 px patch of the pattern therefore identifies its absolute projector location. The network's job is to learn that codebook implicitly and read it through real-world nuisances (defocus blur, oblique foreshortening, albedo texture, sensor noise, shadows) — which a literal window-decoder cannot survive, but a large receptive field can.

3 · Network architecture

A 4-level U-Net (encoder–decoder with skip connections). Every stage is a _Block: two 3×3 convolutions, each followed by GroupNorm (8 channels/group) and SiLU. Downsampling is 2×2 max-pool; upsampling is bilinear ×2 followed by the block after concatenating the matching encoder skip. The head is a single 1×1 convolution.

4 · The classification + offset head

Direct regression of an absolute coordinate over a 1920 px range is ill-conditioned: the gradient only says "larger / smaller", so predictions crawl across the frame (the regression baseline plateaued near 360 px). Instead, the coordinate is factored into a coarse discrete bin (well-conditioned cross-entropy — the argmax can jump anywhere immediately) and a fine continuous offset whose dynamic range is one bin instead of the whole frame — that is where subpixel precision comes from.

Loss (per valid pixel, mask from gt_proj NaNs)

L = CE(u-bin) + CE(v-bin) + 2·( |f̂u − fu| + |f̂v − fv| ) + BCE(validity)

Offsets are supervised at the ground-truth bin (teacher forcing). Chance level for the CE terms is ln 60 + ln 36 ≈ 7.7 — the visible "plateau at 8" before the network breaks through and starts reading the code.

8 · Known limitations & next steps

IssuePlan
Wrong-bin tail (p95 in the hundreds of px), concentrated in blurred / dark / oblique regions more epochs (in progress) · use bin-softmax confidence to extend the validity mask · median/consistency filtering at inference
Median 10 px ≈ ⅓ bin — offset head not yet subpixel (multi-shot Gray code reaches ~0.25 px) raise offset loss weight · soft-argmax over neighboring bins · longer training at low LR
Synthetic-only training (Lambertian direct light, no interreflections / speculars) Mitsuba-rendered validation slice; later fine-tune on real captures
Single fixed pattern (codebook memorized in weights) by design for deployment; correlation/RAFT-style matching is the pattern-agnostic upgrade path

9 · Reproduce

# data (≈4 s/sample, resumable, seeds disjoint per split)
python scripts/gen_training_data.py --n 10000 --seed 100000 --out renders/val \
    --patterns patterns/marray --lean --jobs 8

# pack the memmap loaf
python scripts/build_loaf.py --data renders/val --out renders/val_loaf

# train (resumable; checkpoints best val median)
python scripts/train_proj_net.py --loaf renders/val_loaf --epochs 25 --batch 64 \
    --amp --workers 4 --val 4 --lr 1e-3 --out checkpoints/proj_net.pt

# inference + scoring on any sample folder
python scripts/predict_proj_net.py renders/val/sample_100000
